托管在Github上的Hexo博客怎么样绑定个人域名

使用Hexo搭建了博客之后，大家都会选择把博客托管到Github上的gitpage或者别的平台，此时我们上线博客的默认地址是 xxx.github.io。这是github给我们的一个免费的二级域名。

作为精致主义者，怎么能容忍自己的博客是别人又臭又长的二级域名，今天来说说托管在Github上的Hexo博客怎么样绑定个人域名？

这块教程，网上挺多，但大多数都是写的乱七八糟，又是A记录又是CNAME记录，纯属胡拼乱凑的。

一、代码部署

这块就不细说，简言之就是把我们的博客部署在Github上，访问自己的二级域名 xxx.github.io，正常使用后备用

二、域名准备

既然是绑定自己的域名，那首先就要准备好自己的域名了，可以是顶级的也可以是二级的，例如飞鸟的 note.yaxi.net,没有的话可以去阿里云等平台注册，当然现在也注册不了多好的域名了，想要短位的只能选非主流后缀了，比如me、cc、io等，这边不扯

三、域名解析

既然域名已经准备好了，现在就需要把域名绑定到Hexo博客上了，此时需要打开你域名的DNS解析系统，下面提供两个方法

关于DNS解析不会的可以另行学习

方法一：解析A记录

A记录就是直接将域名解析到ip地址，首先需要获取到我们博客的ip地址，大家可以直接使用CMD命令行工具ping一下我们的博客,如下

得到我们的ip地址，然后在DNS解析系统里添加A记录为上述获得的ip地址，顶级域名记得添加www和@ 两条记录以方便 www.yourdomain.com 和 yourdomain.com 两个都能正常访问；二级域名就直接给xxx.yourdomain.com添加A记录就可以了

方法二：解析CNAME记录（推荐）

上面一个方法，我们直接将域名解析到了博客的ip地址，这有个很大的缺点就是，咱们在github上的博客服务器ip地址有可能会更改的，一旦ip地址更改，我们就需要重新去设置DNS解析。所以飞鸟更推荐下面这个方法

在DNS解析系统设置CNAME解析，值为xxx.github.io（也就是系统给你的二级域名），这样不管ip地址怎么换，域名是永远指向 xxx.github.io的，更换ip这种事就让github去操作就行了。

注意：在这一步DNS解析，网上很多教程写的很乱，有的让用A解析，有的让用CNAME解析，还有的竟然让WWW设置A解析，@设置CNAME解析，简直是不知所云…

四、设置CNAME（重要的一步）

上面我们给域名做了解析，那下面我们就要给服务器端（github）设置域名绑定了，不能说你域名单方面想绑在博客就行了，人家服务器也得任你这个域名啊，必须要在服务器端进行操作，也有两个方法：

1.直接在github操作，

登陆github，打开 xxx.github.io 项目的（Settings）设置，然后在 GitHub Pages的 Custom domain设置里填上购买的域名如下图。

可以勾选一下下面的Enforce HTTPS，可以快速给博客申请https证书，实现博客https访问。

2.使用git提交（推荐）

在本地hexo文件夹，在source文件夹下新建一个文件 CNAME （无后缀），使用文件编辑器打开此文件，在里面填入你的域名，如本站的 note.yaxi.net 保存。
然后部署到github即可

部署之后，系统会自动将xxx.github.io 项目的（Settings）里Custom domain的设置改为你的域名，所以还是直接git提交是最方便的。

然后稍等片刻，域名就算配置完成了，测试正常访问就ok了。
